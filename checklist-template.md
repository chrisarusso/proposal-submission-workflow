# Proposal Submission Checklist Template

This checklist template contains default items for proposal submissions. Each proposal can have additional custom items added.

## Pre-submission [stage:early]

- [ ] [check:completeness] Review RFP/requirements document thoroughly
- [ ] Identify key stakeholders and assign roles
- [ ] Set proposal deadline and internal milestones
- [ ] Create proposal document structure outline
- [ ] Gather required company information and credentials
- [ ] Review past similar proposals for reference

## Document Preparation [stage:early] [stage:mid]

- [ ] [check:completeness] Write executive summary section
- [ ] [check:completeness] Develop project approach/methodology section
- [ ] [check:completeness] Create team qualifications and bios section
- [ ] [check:completeness] Prepare budget and pricing section
- [ ] [check:completeness] Include timeline and project schedule
- [ ] [check:completeness] Add company background and experience
- [ ] [check:completeness] Include case studies or relevant examples
- [ ] [check:strings:NDA,confidentiality] Ensure required legal terms are included
- [ ] Format document according to requirements
- [ ] Add table of contents and page numbers

## Review & Approval [stage:mid] [stage:late]

- [ ] Internal technical review completed
- [ ] Budget review and approval obtained
- [ ] Legal/compliance review completed
- [ ] Executive review and sign-off obtained
- [ ] [check:spelling] Final proofread of proposal document
- [ ] [check:completeness] Verify all required sections are present
- [ ] Verify all attachments and appendices included
- [ ] Check formatting and branding consistency

## Submission [stage:late]

- [ ] [check:strings] Verify all custom required strings are present
- [ ] Final validation run (validateBeforeSubmission)
- [ ] Convert to required format (PDF, Word, etc.)
- [ ] Submit proposal through required channel
- [ ] Confirm receipt with client/RFP issuer
- [ ] Update proposal status to "Submitted"
- [ ] Archive proposal documents

## Notes

- Items marked with `[check:spelling]` will trigger LLM-based spelling/grammar checks
- Items marked with `[check:completeness]` will verify required sections are present
- Items marked with `[check:strings:keyword1,keyword2]` will check for required strings
- Stage indicators `[stage:early]`, `[stage:mid]`, `[stage:late]` determine when automated checks run
- Custom checklist items can be added per proposal via the UI
- Custom required strings can be added per proposal and will be validated before submission
